79  /*
80  * Splits the first node of the list in two nodes, that become the
81  * children of the node.
82  *
83  * The zone is split vertically if node->height > node->width,
84  * horizontally otherwise, in order to minimize the size of
85  * frontiers.
86  *
87  * By construction, the leafs are ordered so that largest zones are
88  * at the beginning of the list, so its consistent to split those
89  * nodes first. Indeed, split nodes are always pushed back in the
90  * list, while the first element of leafs is popped. Moreover, at
91  * the beginning of the algorithm, the only element in `leafs` is
92  * `root`, that represents the whole grid.
93  *
94  * Also note that this procedure increases the leafs size by 1: 2
95  * children are added at the end of the list, while the split node
96  * is removed from the beginning of the list.
97  */
