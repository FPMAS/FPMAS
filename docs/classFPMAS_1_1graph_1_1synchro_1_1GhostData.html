Detailed Description

template<class T>
class FPMAS::graph::synchro::GhostData< T >

Synchronisation mode used as default by the DistributedGraph, and wrapper for distant data represented as "ghosts".

In this mode, overlapping zones (represented as a "ghost graph") are built and synchronized on each proc.

When a node is exported, ghost arcs and nodes are built to keep consistency across processes, and ghost nodes data is fetched at each GhostGraph::synchronize() call.

Getters used are actually the defaults provided by SyncData and LocalData, because once data has been fetched from other procs, it can be accessed locally.

In consequence, such "ghost" data can be read and written as any other LocalData. However, modifications won't be reported to the origin proc, and will be overriden at the next synchronization.
