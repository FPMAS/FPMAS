◆ FPMAS_ID_TYPE

#define FPMAS_ID_TYPE   unsigned long

Type used to represent the "id" part of an fpmas::api::graph::DistributedId.

This type defines the maximum count of Nodes and Edges that might be created during a single simulation. (e.g. 65535 for a 16 bit unsigned integer)

It can be user defined at compile time using cmake -DFPMAS_ID_TYPE=<unsigned integer type> ..

By default, the unsigned long type is used, that can be 32 or 64 bit depending on systems.

Any unsigned integer type can be specified. The std::uintN_t like types defined in the C++ standard can notably be used to fix the FPMAS_ID_TYPE size independently of the underlying system.

FPMAS currently supports 16, 32 and 64 unsigned integer types.
