46  /*
47  * About the comm_references usage :
48  * The MPI reference only defines the MPI_Comm_dup function to copy
49  * communicators, what requires MPI calls on all other procs to be performed. In
50  * consequence, MPI_Comm_dup is :
51  * 1- Oversized for our need
52  * 2- Impossible to use in practice, because all procs would have to sync each
53  * time a copy constructor is called...
54  *
55  * Moreover, it is safe in our context for all the copied / re-assigned
56  * MpiCommunicator from this instance to use the same MPI_Comm instance.
57  * However, calling MPI_Comm_free multiple time on the same MPI_Comm instance
58  * results in erroneous MPI memory usage (this has been verified with
59  * valgrind). In consequence, we use the comm_references variable to smartly
60  * manage the original MPI_Comm instance, in order to free it only when all
61  * the MpiCommunicator instances using it have been destructed.
62  */
63  int comm_references = 1;
