124  /*
125  * Using SyncData<T>* pointers directly has Node data in the
126  * DistributedGraph (and in the GhostGraph) would require extra memory
127  * management and storage at the Graph scale to manually delete (and copy, and
128  * assign, etc) node data. Using this smart pointer instead avoid this,
129  * because its destructor is automatically and properly called as
130  * defined at the Graph scale as nodes are created or deleted.

132  * have been optimal.
