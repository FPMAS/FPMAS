31  /*
32  * About the comm_references usage :
33  * The MPI reference only defines the MPI_Comm_dup function to copy
34  * communicators, what requires MPI calls on all other procs to be performed. In
35  * consequence, MPI_Comm_dup is :
36  * 1- Oversized for our need
37  * 2- Impossible to use in practice, because all procs would have to sync each
38  * time a copy constructor is called...
39  *
40  * Moreover, it is safe in our context for all the copied / re-assigned
41  * MpiCommunicator from this instance to use the same MPI_Comm instance.
42  * However, calling MPI_Comm_free multiple time on the same MPI_Comm instance
43  * results in erroneous MPI memory usage (this has been verified with
44  * valgrind). In consequence, we use the comm_references variable to smartly
45  * manage the original MPI_Comm instance, in order to free it only when all
46  * the MpiCommunicator instances using it have been destructed.
47  */
48  int comm_references = 1;
