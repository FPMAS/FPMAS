Detailed Description

Generic Agent API.

Agents is a simulation are assumed to take advantage of polymorphism : a single type is used to store Agents in an api::graph::DistributedGraph (more precisely, AgentPtr is used), but pointed Agents might have different types and implement different the act() functions to produce different behaviors. Such considerations implies a few constraints :

  • each agent type must be associated to an unique TypeId, so that the underlying type of each Agent can be deduced thanks to the method typeId().
  • a copy() method must be implemented to dynamically allocate copies of Agents, with the same underlying type, but directly from this generic API.
