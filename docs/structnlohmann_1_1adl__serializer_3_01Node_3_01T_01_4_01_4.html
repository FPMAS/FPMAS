Defines rules to deserialize a Node from a JSON string.

The following JSON format can be used to unserialize nodes :

{
"id":"<node_id>",
"data":<json_data_representation>,
"weight":<node_weight>
}

data and weight fields are both optional, with a default weight value set to 1.

If a data field is specified, the Node data will be instanciated dynamically using a new statement and the copy constructor of the specified data type T.

Some, in this case, memory will internally be allocated by the library, so it is the responsability of the user to free the data field.

To avoid this, the data field can be ignored and it is still possible to manually associate data to nodes using the Node::setData(T*) function.

Defines rules to serialize node instances in the following general JSON format :

{
"id":"<node_id>",
"data":<json_data_representation>,
"weight":<node_weight>
}

The huge interest of this json implementation is that the JSON for Modern C++ library will automatically calls the appropriate functions to serialize the data field, whatever its type is. In consecuence, users can specify their own to_json function to serialize their data type, and any node (and so any graph) can be serialized properly without altering or even know about this internal node and graph serialization process.

The to_json(json&, const Graph<T>&) implicitly uses this function to serialize graphs.
