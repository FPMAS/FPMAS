const FPMAS::graph::Graph< T > &  graph

This function defines rules to implicitly serialize graphs as json strings using the JSON for Modern C++ library. For more information about how to serialize objects, see the JSON library documentation and the small example given in the Graph class documentation.

The general JSON (prettified) format used to represent graph is as follow :

{
"arcs":[
...
{"id":"<arc_id>","link":["<source_node_id>","<target_node_id>"]},
...
],
"nodes":[
...
{"id":"<node_id>","data":<json_data_representation>},
...
]
}

Notice that the real serialized version does not contains any indentation, line breaks or spaces.

Any custom data type can be easily serialized, as explained in the to_json(json&, const Node<T>&) function, allowing the user to serialize any custom graph without having to know about the internal serialization processes.

Note that a reference to the graph object, not a reference to the pointer, must be used as argument.

Graph<T> g = new Graph<T>();
// Do processing to build the graph
// This WON'T WORK
json j = g;
// This is ok
json j = *g;
delete g;

template<class T >
void FPMAS::graph::to_json ( json &  j,
const FPMAS::graph::Node< T > &  n 
)

Defines rules to serialize node instances in the following general JSON format :

{"id":"<node_id>", "data":<json_data_representation>}

The huge interest of this json implementation is that the JSON for Modern C++ library will automatically calls the appropriate functions to serialize the data field, whatever its type is. In consecuence, users can specify their own to_json function to serialize their data type, and any node (and so any graph) can be serialized properly without altering or even know about this internal node and graph serialization process.

The to_json(json&, const Graph<T>&) implicitly uses this function to serialize graphs.
